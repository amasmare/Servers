نموذج OSI السباعي الطبقات

نموذج OSI (Open Systems Interconnection) يقسّم الاتصالات الشبكية إلى 7 طبقات. انقر على كل طبقة لعرض تفاصيلها والبروتوكولات المرتبطة بها.

🔍 الطبقات السبع — انقر للاستكشاف

7
طبقة التطبيق — Application Layer
واجهة المستخدم مع الشبكة. توفر خدمات الشبكة للتطبيقات مثل المتصفحات والبريد الإلكتروني. تتعامل مع البيانات التي ينشئها المستخدم مباشرةً.
بروتوكولات: HTTP, HTTPS, FTP, SMTP, DNS, DHCP, SSH, Telnet, SNMP
6
طبقة التقديم — Presentation Layer
مترجم البيانات. تتولى تشفير البيانات وضغطها وتحويل صيغها. تضمن أن البيانات المرسلة من نظام معين يمكن قراءتها بواسطة النظام المستقبِل.
تشفير SSL/TLS، ضغط JPEG/MPEG، ترميز ASCII/Unicode
5
طبقة الجلسة — Session Layer
مدير الجلسات. تنشئ جلسات الاتصال وتديرها وتنهيها. تتحكم في الحوار بين الجهازين وتتعامل مع التزامن ومعالجة الأخطاء على مستوى الجلسة.
بروتوكولات: NetBIOS, RPC, SIP, SQL, NFS
4
طبقة النقل — Transport Layer
مسؤول تسليم البيانات. توفر النقل الموثوق (TCP) أو السريع (UDP) من نقطة لأخرى. تتعامل مع تقسيم البيانات إلى مقاطع، والتحكم في التدفق، واسترداد الأخطاء. الوحدة: Segment.
بروتوكولات: TCP (موثوق، اتصالي) ، UDP (سريع، لا اتصالي)
3
طبقة الشبكة — Network Layer
مخطط المسار. مسؤولة عن عنونة IP والتوجيه (Routing) بين الشبكات المختلفة. تحدد أفضل مسار لنقل البيانات من المصدر إلى الوجهة عبر شبكات متعددة. الوحدة: Packet.
بروتوكولات: IP (IPv4/IPv6)، ICMP، ARP، OSPF، BGP
2
طبقة ربط البيانات — Data Link Layer
مدير الاتصال المحلي. تتحكم في وصول الوسائط (MAC) وتكتشف أخطاء البيانات على مستوى الشبكة المحلية. تتعامل مع عناوين MAC الفيزيائية. الوحدة: Frame. تنقسم إلى: LLC (التحكم في الارتباط المنطقي) و MAC (التحكم في الوصول للوسائط).
بروتوكولات/تقنيات: Ethernet، Wi-Fi (802.11)، VLAN (802.1Q)، PPP
1
الطبقة المادية — Physical Layer
الوسط الفيزيائي الفعلي. تتعامل مع الإشارات الكهربائية والضوئية واللاسلكية. تحدد مواصفات الكابلات والموصلات والإشارات ومعدلات البيانات. الوحدة: Bit.
تقنيات: UTP/STP، Fiber Optic، Wi-Fi، Bluetooth، RJ-45، SC/LC connectors

تعرّف على أنواع الكابلات المستخدمة في شبكات الخوادم، مواصفاتها، ومتى تستخدم كل نوع.

UTP (Unshielded Twisted Pair) — الأكثر شيوعاً، غير معزول، أرخص.

💡 الكابلات الضوئية (Fiber Optic)

SMF — Single-Mode Fiber
• نواة صغيرة جداً (8-10 ميكرون)
• مسافات طويلة جداً (حتى 100 كم+)
• ليزر أصفر/أزرق
• للاتصالات بين المواقع والـ WAN
MMF — Multi-Mode Fiber
• نواة أكبر (50-62.5 ميكرون)
• مسافات متوسطة (حتى 2 كم)
• LED أو ليزر برتقالي/فيروزي
• للشبكات داخل المباني والحرم الجامعي
موصلات الألياف الضوئية:
ST (Straight-Tip) — موصل دوّار قديم للـ MMF.
SC (Subscriber Connector) — موصل مربع الشكل، يُدفع للداخل.
LC (Lucent Connector) — موصل صغير الحجم، الأكثر شيوعاً في مراكز البيانات.
SFP/SFP+ — وحدة استقبال-إرسال صغيرة قابلة للإدخال في مقبس GBIC.

❓ اختبار الكابلات السريع

عنونة IPv4 والشبكات الفرعية

عنوان IPv4 مكوّن من 32 بتًا (4 أوكتيتات). يُقسَّم إلى جزء الشبكة (Network) وجزء المضيف (Host) باستخدام قناع الشبكة الفرعية (Subnet Mask).

🔢 حاسبة الشبكات الفرعية (CIDR)

أدخل عنوان IP مع بادئة CIDR (مثل: 192.168.1.100/24) لحساب معلومات الشبكة.

📊 فئات العناوين الخاصة (RFC 1918)

النطاققناع الشبكة الافتراضيعدد الشبكاتالاستخدام
10.0.0.0 – 10.255.255.255/81شبكات المؤسسات الكبيرة
172.16.0.0 – 172.31.255.255/1216شبكات المؤسسات المتوسطة
192.168.0.0 – 192.168.255.255/16256الشبكات المنزلية والصغيرة
APIPA: 169.254.x.x — يُعيَّن تلقائياً عند فشل DHCP. يعني أن الجهاز لا يستطيع الوصول للشبكة.
Loopback: 127.0.0.1 — عنوان الاختبار الداخلي (localhost).
NAT/PAT: ترجمة العناوين — تخفي العناوين الخاصة خلف IP عام واحد.

🌐 مقدمة IPv6

الحجم: 128 بتًا (مقابل 32 بتًا في IPv4) — يكفي لـ 340 أُندِسيليون عنوان.
الصيغة: 8 مجموعات هكساديسيمالية مفصولة بنقطتين — مثال: 2001:0db8:85a3::8a2e:0370:7334
تقصير: يمكن إسقاط الأصفار الأمامية وإحلال :: بدل مجموعات الأصفار المتتالية (مرة واحدة فقط).
Link-Local: fe80::/10 — يُعيَّن تلقائياً لكل واجهة شبكة.
Loopback: ::1
تقنيات الانتقال: 6to4، ISATAP، Teredo — تتيح تشغيل IPv6 فوق شبكات IPv4.

TCP مقابل UDP والمنافذ المشهورة

المنافذ (Ports) تعرّف خدمة الشبكة. المنافذ المعروفة (Well-Known) هي 0-1023. فهم الفرق بين TCP و UDP ضروري للإعداد الصحيح لجدران الحماية والخوادم.

⚖️ TCP مقابل UDP

TCP — موثوق واتصالي
✅ مصافحة ثلاثية (SYN → SYN-ACK → ACK)
✅ ضمان وصول الحزم بالترتيب
✅ إعادة إرسال الحزم الضائعة
✅ التحكم في التدفق (Flow Control)
❌ أبطأ من UDP بسبب الحمل الإضافي
استخدام: HTTP, HTTPS, FTP, SSH, Email
UDP — سريع ولا اتصالي
✅ لا مصافحة — يرسل مباشرة
✅ أسرع بكثير من TCP
✅ أقل حملاً على الشبكة
❌ لا ضمان لوصول الحزم
❌ لا ترتيب مضمون للحزم
استخدام: DNS, DHCP, VoIP, فيديو حي, DNS
المصافحة الثلاثية TCP (Three-Way Handshake):
1. العميل يرسل SYN (طلب الاتصال)
2. الخادم يرد بـ SYN-ACK (تأكيد القبول)
3. العميل يرسل ACK (تأكيد الاستلام) — الاتصال مُنشأ

DHCP و DNS و WINS

هذه الخدمات أساسية في أي شبكة خوادم. DHCP يوزّع عناوين IP تلقائياً، DNS يترجم أسماء النطاقات، وWINS يحل أسماء NetBIOS.

🔄 DHCP — تبادل الحزم الأربع (DORA)

يتفاوض العميل مع خادم DHCP عبر 4 رسائل. انقر كل خطوة لعرض التفاصيل.

D
Discover — اكتشاف
العميل يبث رسالة على المنفذ UDP 67 بحثاً عن أي خادم DHCP متاح. المصدر: 0.0.0.0 — الوجهة: 255.255.255.255 (بث عام).
نوع الرسالة: UDP Broadcast ← العميل
O
Offer — عرض
خادم DHCP يعرض عنوان IP محجوزاً مؤقتاً مع معلومات الشبكة (قناع الشبكة، البوابة الافتراضية، DNS، مدة الإيجار). يُرسَل إلى عنوان MAC الخاص بالعميل.
نوع الرسالة: UDP Unicast/Broadcast ← الخادم
R
Request — طلب
العميل يقبل العرض ويطلب رسمياً استخدام العنوان المعروض. يُرسَل كبثّ (Broadcast) حتى يعلم خوادم DHCP الأخرى أن عرضها رُفض.
نوع الرسالة: UDP Broadcast ← العميل
A
Acknowledgment — تأكيد
الخادم يؤكد تخصيص العنوان للعميل بشكل رسمي ويرسل جميع معلومات الإعداد النهائية. يبدأ العميل بعدها باستخدام العنوان رسمياً.
نوع الرسالة: UDP Unicast/Broadcast ← الخادم
APIPA (169.254.x.x): إذا فشل الاتصال بخادم DHCP، يعيّن ويندوز عنوان APIPA تلقائياً (169.254.1.0 – 169.254.254.255). هذا يعني عدم وجود اتصال بالشبكة الخارجية.

🌍 DNS — نظام أسماء النطاقات

الهرمية: Root (.) → TLD (.com, .org) → Domain (google.com) → Subdomain (mail.google.com)
FQDN (Fully Qualified Domain Name): الاسم الكامل المؤهل — مثال: www.example.com. (مع نقطة في النهاية)
Forward Lookup: اسم النطاق → عنوان IP (السجل A أو AAAA)
Reverse Lookup: عنوان IP → اسم النطاق (سجل PTR)
سجلات DNS الرئيسية:
نوع السجلالوظيفةمثال
Aاسم نطاق → IPv4example.com → 93.184.216.34
AAAAاسم نطاق → IPv6example.com → 2606:2800::1
CNAMEاسم نطاق → اسم آخر (مستعار)www → example.com
MXخادم البريد للنطاقmail.example.com
PTRIP → اسم نطاق (عكسي)34.216.184.93.in-addr.arpa
SOAمعلومات منطقة DNS الأصلية—
WINS (Windows Internet Name Service): يحل أسماء NetBIOS إلى عناوين IP. يُستخدم في شبكات Windows القديمة. مهجور في الشبكات الحديثة لصالح DNS.
