🔑 PKI Key Concepts Quick Reference

CA (Certificate Authority)
Trusted entity that issues and signs digital certificates. Root CA → Intermediate CA → End-entity cert.
Private Key
Secret key kept only on the server. Used to decrypt data and sign messages. Never shared.
Public Key
Shared in the certificate. Anyone can use it to encrypt data for the server or verify signatures.
CSR
Certificate Signing Request — server sends public key + identity info to CA requesting a certificate.
CRL / OCSP
Certificate Revocation List / Online Certificate Status Protocol — check if a cert has been revoked.
SSL/TLS
Protocol using PKI to encrypt communication. TLS 1.3 is current standard. Uses certificates for identity.
