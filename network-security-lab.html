💡 Firewall Best Practices

• Deny all by default — implicit deny as the last rule
• Allow only what's needed — whitelist approach
• Most specific rules first — rules are read top-to-bottom
• Block management ports (22, 3389) from internet — limit to management VLAN
• Stateful inspection — track connection state, not just packets
• Log denied traffic — essential for incident investigation

VPN & IPsec

Learn the different VPN types and IPsec modes. Click each scenario to identify the best solution.

🔒 IPsec Modes

Transport Mode
Encrypts only the payload (data) of IP packets. Original IP headers are preserved. Used for host-to-host communications.
[IP Header] [ESP Header] [Encrypted Payload]
Tunnel Mode
Encrypts the entire original IP packet and adds a new outer IP header. Used for network-to-network VPN tunnels (site-to-site).
[New IP Header] [ESP Header] [Encrypted (Original IP + Payload)]

🌐 VPN Types

Site-to-Site VPN
Connects entire networks (office to data center). Always-on, transparent to users. IPsec Tunnel mode.
Remote Access VPN (SSL/TLS)
Individual users connect to corporate network. Client software or browser-based. Cisco AnyConnect, OpenVPN.
Split Tunneling
VPN only routes corporate traffic through tunnel; internet goes direct. Reduces VPN load but reduces security.
Full Tunnel VPN
ALL traffic routed through VPN. Maximum visibility and control. Higher bandwidth usage on VPN server.
